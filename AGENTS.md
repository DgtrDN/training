# AGENTS — Инструкции для AI-кодовых агентов

Короткие, практичные указания, чтобы агент быстро стал продуктивен в этом репозитории.

## Что в репозитории
- Проект содержит одну тетрадь: [training.ipynb](training.ipynb).

## Что агент может и должен делать
- **Открыть тетрадь**: изучить содержимое и структуру ячеек в [training.ipynb](training.ipynb).
- **Запуск кода**: если требуется выполнить код, создавать или предлагать команды для запуска ячеек в локальной Python-среде.
- **Среда исполнения**: предполагать стандартную среду Python 3.8+; предлагать создание виртуального окружения (venv/conda) и `requirements.txt`, если нужно установить зависимости.
- **Не портить файл**: при предложении изменений для тетради — давать патчи в виде .py-версий или инструкций по обновлению ячеек, не перезаписывать тетрадь без уточнения.

## Чего нет в репозитории
- Нет скриптов сборки, тестов или CI-конфигурации. Если предлагается добавить — сначала согласовать с владельцем репозитория.

## Советы агенту
- Следовать принципу «связывать, а не встраивать»: ссылаться на существующие файлы и документацию.
- Быть кратким и предлагать минимальные изменения.
- Если нужен запуск или тестирование — сначала вывести команды, которые пользователь может скопировать и выполнить локально.

## Предложенные следующиe шаги (опционально)
- Создать `.github/copilot-instructions.md` для рабочих инструкций, если репозиторий вырастет.
- Добавить `requirements.txt` и простой `run.sh` для воспроизводимости среды.
